Data Encryption and Key Management

How Relancy protects data at rest and in transit using modern encryption standards.

Back to Knowledge Base

Overview

Relancy uses industry-standard encryption to protect sensitive data. Encryption is applied at rest, in transit, and at the application layer where appropriate.

Encryption at rest

All tenant data is encrypted using strong algorithms such as AES-256. Storage is isolated per tenant.

Encryption in transit

All communication between clients, APIs, and storage uses TLS 1.2+ to prevent interception or tampering.

Key management

  • Keys are rotated regularly
  • Keys are stored in secure key vaults
  • Access to keys is restricted and audited
  • Tenant-specific keys may be used depending on plan

Audit and compliance

All key access and encryption-related operations are logged for compliance and security review.

Private Deployment options

Private Deployment tenants may bring their own keys (BYOK) or use dedicated key vaults.
