Tenant Customization Boundaries

What tenants can customize—and what remains platform-controlled.

Back to Knowledge Base

Overview

Relancy allows tenants to tailor the platform to their operational needs while preserving lifecycle discipline, audit integrity, and platform consistency. This article outlines what can be customized in the shared SaaS environment and what requires Private Deployment.

Allowed customizations (SaaS)

Custom fields

Tenants can add custom fields to cases, incidents, investigations, and people. These fields:

  • Are tenant-specific
  • Do not affect platform behavior
  • Appear in reporting and exports

Panels and contextual displays

Panels allow tenants to surface external information or contextual data without altering lifecycle logic.

Webhooks

Tenants can subscribe to lifecycle events and build external automations that react to platform activity.

Branding

Basic branding options such as logo and color accents may be available depending on the tenant’s plan.

Restricted customizations (SaaS)

To protect platform integrity, the following are not allowed in the shared SaaS environment:

  • Custom logic or code execution
  • Workflow overrides
  • Write-level API access
  • Custom lifecycle stages
  • Direct database access
  • Dedicated API clusters

Customizations available in Private Deployment

Organizations requiring deeper customization can use the Private Deployment tier, which supports:

  • Custom logic and workflow extensions
  • Write-level APIs
  • Dedicated infrastructure
  • Custom lifecycle augmentations
  • Tenant-specific deployment schedules
