When to Choose Private Deployment

Understanding when dedicated infrastructure is the right fit.

Back to Knowledge Base

Overview

Relancy’s shared SaaS environment meets the needs of most organizations. However, some require deeper control, custom logic, or dedicated infrastructure. This article helps you determine whether Private Deployment is the right choice.

Choose Private Deployment if you need:

1. Write-level API access

If your organization must write directly into Relancy’s entities or extend lifecycle behavior, Private Deployment provides the necessary control.

2. Custom logic or workflow extensions

Private Deployment allows tenant-specific logic that would not be safe in a shared environment.

3. Dedicated API clusters

Organizations with strict performance, isolation, or regulatory requirements may need their own load- balanced API infrastructure.

4. Sovereign or air-gapped environments

Private Deployment supports VPC, on-prem, and sovereign cloud scenarios.

5. Custom deployment cadence

If you require control over update timing or change windows, Private Deployment offers coordinated releases.

When SaaS is sufficient

The SaaS tier is ideal when you need:

  • Zero maintenance
  • Automatic updates
  • Strong tenant isolation
  • Predictable costs
  • Audit-first lifecycle enforcement

Cost considerations

Private Deployment is a premium offering due to dedicated infrastructure, custom SLAs, and increased operational responsibility. Organizations should evaluate whether the additional control justifies the investment.
