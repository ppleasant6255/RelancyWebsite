Session Management Architecture

How Relancy manages user sessions using secure tokens, expiration rules, and activity monitoring.

Back to Knowledge Base

Architecture Overview

Session management uses secure, short-lived tokens issued by the identity provider. Sessions are monitored for activity, expiration, and anomalous behavior.

Session controls

  • Idle timeout enforcement
  • Absolute session expiration
  • Token refresh rules
  • Concurrent session limits
  • Forced logout for security events

Operational safeguards

Suspicious session activity triggers alerts and automatic session termination. Privileged sessions require shorter timeouts.

Audit integrity

Session events include timestamp, IP metadata (if allowed), device type, and token lifecycle details.
