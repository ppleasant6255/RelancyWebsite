Isolation Overview

Relancy is designed with strict multi‑tenant isolation to ensure that tenant data, workflows, and configuration remain completely segregated. Isolation applies across storage, authentication, and operational layers.

Isolation layers

  • Data storage isolation
  • Authentication and RBAC isolation
  • API request scoping
  • Webhook and integration isolation
  • Tenant‑specific encryption keys

Operational safeguards

  • No cross‑tenant queries
  • No shared caches for sensitive data
  • Strict validation of tenant context on every request
  • Tenant‑scoped audit logs

Private Deployment

Private Deployment tenants receive dedicated infrastructure, eliminating multi‑tenant concerns entirely.
