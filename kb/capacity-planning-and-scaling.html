Scaling Overview

Relancy uses elastic scaling to maintain performance during peak usage. Capacity planning ensures that compute, storage, and network resources remain sufficient for all tenants.

Scaling triggers

  • High API throughput
  • Background job backlog
  • Increased notification volume
  • Large data imports or exports
  • Tenant onboarding surges

Operational safeguards

  • Automatic horizontal scaling
  • Load‑balanced API endpoints
  • Isolated tenant resource pools
  • Real‑time performance monitoring

Audit integrity

Scaling events and resource allocation changes are logged for operational and compliance review.
