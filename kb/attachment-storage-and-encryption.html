Attachment Storage & Encryption

How Relancy stores, encrypts, and secures attachments across incidents, cases, and investigations.

Back to Knowledge Base

Storage Overview

Attachments—including documents, images, audio, and video—are stored in encrypted, tenant‑isolated storage. Access is governed by RBAC and audit logging.

Encryption rules

  • Encryption at rest using tenant‑scoped keys
  • Encryption in transit using secure protocols
  • Hashing for integrity verification
  • Strict access validation on every request

Operational safeguards

Attachments cannot be modified after upload. Redactions create new versions while preserving the original.

Audit integrity

All attachment uploads, views, and redactions are logged for compliance and defensibility.
