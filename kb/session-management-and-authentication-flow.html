Authentication Overview

Relancy uses secure, token‑based authentication to validate user identity and enforce RBAC permissions across the platform. Sessions are short‑lived and automatically refreshed.

Session lifecycle

  • Initial login via secure identity provider
  • Short‑lived access tokens for API calls
  • Automatic silent token refresh
  • Session expiration after inactivity

Security controls

  • Encrypted token storage
  • Device‑bound session validation
  • Automatic logout on token compromise
  • Strict RBAC enforcement on every request

Audit logging

Login attempts, token refreshes, and session expirations are logged for security monitoring.
