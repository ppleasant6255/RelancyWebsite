Rate Limiting Overview

API rate limits prevent abuse, ensure fair resource usage, and protect system stability. Limits apply per tenant, per endpoint, and per integration key.

Rate limit categories

  • Per-minute request limits
  • Burst protection
  • Tenant-level quotas
  • Integration-specific throttles
  • Webhook delivery rate limits

Operational safeguards

Exceeding limits triggers throttling and alerts. Critical integrations receive priority lanes with elevated thresholds.

Audit integrity

Rate limit violations and throttling events are logged with timestamp, endpoint, and tenant metadata.
