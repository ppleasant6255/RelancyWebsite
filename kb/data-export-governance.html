Data Export Governance

How Relancy ensures safe, compliant handling of exported data.

Back to Knowledge Base

Overview

Data exports provide powerful insights but must be handled responsibly. Relancy enforces governance rules to ensure that exported data remains secure, compliant, and auditable.

Export types

  • CSV exports
  • Report downloads
  • API-based data pulls
  • Webhook-delivered event data

Governance controls

  • Role-based export permissions
  • Audit logging of all exports
  • Tenant-scoped data boundaries
  • Optional export watermarking

Compliance considerations

Exported data may contain sensitive or protected-class information. Organizations should apply internal governance policies to ensure safe handling.

API governance

API-based exports are rate-limited and authenticated to prevent misuse or excessive data extraction.
