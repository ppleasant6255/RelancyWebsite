Storage Overview

Relancy uses a multi‑tenant storage model that isolates each tenant’s data into separate encrypted partitions. Storage is horizontally scalable and optimized for high‑volume workloads.

Storage components

  • Tenant‑scoped blob containers
  • Encrypted database partitions
  • Isolated audit log streams
  • Tenant‑specific encryption keys

Operational safeguards

Storage access is validated on every request. Cross‑tenant access is cryptographically impossible due to key isolation.

Audit integrity

Storage access events are logged with timestamp, actor, and partition metadata for compliance review.
