Authentication Overview

Relancy APIs use secure, token‑based authentication to validate identity, enforce RBAC, and ensure tenant isolation. Tokens are short‑lived and must be refreshed regularly.

Token types

  • Access tokens (short‑lived)
  • Refresh tokens (long‑lived, securely stored)
  • Tenant‑scoped API keys (optional for Private Deployment)

Security controls

  • Encrypted token storage
  • Device‑bound session validation
  • Strict tenant scoping on every request
  • Automatic revocation on compromise

RBAC enforcement

API responses are filtered based on the caller’s role and permissions. Unauthorized fields are omitted.

Audit logging

All API authentication events—including token issuance, refresh, and revocation—are logged for security review.
