SaaS vs Private Deployment

Choosing the right operational model for your organization.

Back to Knowledge Base

Overview

Relancy is designed as a secure, audit-first, multi-tenant SaaS platform. Every tenant benefits from the same hardened operational contract, lifecycle discipline, and platform integrity. Some organizations, however, require deeper control, dedicated infrastructure, or custom scaling. For those cases, Relancy offers a Private Deployment tier.

This article explains the difference so you can choose the model that fits your operational needs.

Relancy SaaS (Shared API, Isolated Data)

Principle: Shared worldview, isolated risk.

In the SaaS tier, all tenants run on the same, fully managed API layer. This ensures:

  • Consistent lifecycle behavior
  • Predictable audit trails
  • Uniform security posture
  • Zero drift across tenants
  • Rapid updates and improvements

Behind that shared API, every tenant receives:

  • An isolated database
  • Isolated storage
  • Isolated audit logs
  • Isolated encryption boundary

Your data is never co-mingled with other tenants. Your audit boundary is yours alone.

What SaaS tenants can do

  • Receive webhooks for lifecycle events
  • Export and report on their data
  • Add custom fields and panels
  • Integrate external systems through safe, read-only surfaces
  • Benefit from continuous improvements with no maintenance burden

What SaaS tenants cannot do

To protect audit integrity and lifecycle discipline, SaaS tenants cannot:

  • Write custom logic into the platform
  • Override workflows or lifecycle events
  • Deploy custom code
  • Run their own API cluster
  • Modify core entities
  • Bypass platform rules

Private Deployment (Dedicated Infrastructure, Full Control)

Some organizations require deeper control over their environment. For those cases, Relancy offers a Private Deployment tier. This is not shared SaaS. It is your own Relancy environment—isolated, dedicated, and customizable.

What Private Deployment includes

  • Dedicated API cluster and load balancer
  • Dedicated database and storage
  • Custom scaling rules
  • Custom SLAs
  • Optional write-level APIs
  • Optional custom logic
  • Optional on-prem or VPC deployment

Private Deployment is ideal when you need:

  • Full infrastructure control
  • Custom integrations and lifecycle extensions
  • Strict regulatory or sovereign isolation
  • Air-gapped or highly controlled environments

Because dedicated infrastructure carries higher operational and support cost, Private Deployment is priced as a premium offering.

Which model should you choose?

Choose SaaS if you want:

  • Zero maintenance
  • Automatic updates
  • Predictable costs
  • Strong tenant isolation
  • Audit-first lifecycle discipline
  • A secure, managed environment

Choose Private Deployment if you need:

  • Your own API cluster and scaling rules
  • Your own infrastructure and deployment cadence
  • Custom logic or write-level access
  • Regulatory or sovereign isolation
  • On-prem or VPC deployment
