Offline Mode Behavior

How Relancy operates when devices lose connectivity, including caching, sync rules, and data safety.

Back to Knowledge Base

Offline Overview

Offline mode allows users to continue working when connectivity is unavailable. Relancy caches safe data locally and syncs changes when the device reconnects.

Offline capabilities

  • View cached records
  • Create draft incidents or notes
  • Complete training modules (when supported)
  • Queue updates for later sync

Restrictions

Sensitive or restricted data is never cached offline. Certain workflows require online validation.

Sync behavior

Sync occurs automatically when connectivity returns. Conflicts are resolved using deterministic rules.

Audit integrity

Offline actions are timestamped locally and reconciled with server logs during sync.
