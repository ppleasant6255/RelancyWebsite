Key Management Overview

Relancy uses a centralized, secure key management system to generate, rotate, and store encryption keys. Keys are tenant‑scoped to ensure strict data isolation.

Key types

  • Data encryption keys (DEKs)
  • Key encryption keys (KEKs)
  • Transport encryption keys
  • Tenant‑specific encryption keys

Rotation policies

Keys are rotated automatically on a scheduled basis or immediately upon security events. Rotation is seamless and does not disrupt tenant operations.

Operational safeguards

  • Keys stored in secure hardware modules
  • Strict access controls
  • Audit logging for all key operations
  • Automatic revocation on compromise

Audit integrity

All key generation, rotation, and access events are logged for compliance and security review.
