Multi-tenant Architecture Overview

How Relancy delivers shared efficiency with strict tenant isolation.

Back to Knowledge Base

Overview

Relancy is built on a modern multi-tenant architecture designed to balance efficiency, security, and operational clarity. Tenants share the application layer but operate on isolated data boundaries, ensuring that each organization benefits from the platform’s capabilities without compromising privacy or integrity.

Shared application layer

All tenants use the same API and application logic. This ensures:

  • Consistent lifecycle behavior
  • Uniform security enforcement
  • Predictable updates and improvements
  • Reduced operational overhead

Isolated data layer

Each tenant’s data is stored in an isolated database or schema. This prevents:

  • Cross-tenant data access
  • Audit contamination
  • Data leakage or accidental exposure

Isolation also allows independent scaling and backup strategies.

Tenant context routing

Every request includes tenant context, which the platform uses to:

  • Select the correct database
  • Apply tenant-specific permissions
  • Scope audit entries
  • Enforce isolation rules

Benefits of this architecture

  • Security: Strong data boundaries.
  • Performance: Efficient resource usage.
  • Scalability: Independent tenant scaling.
  • Reliability: Issues in one tenant do not affect others.

Private Deployment option

Organizations requiring dedicated infrastructure can choose the Private Deployment tier, which provides isolated compute, storage, and API clusters.
