Security Overview

Relancy APIs follow industry‑standard security practices to protect tenant data and ensure safe integration with external systems.

Best practices

  • Use short‑lived access tokens
  • Rotate API keys regularly
  • Validate payload signatures
  • Implement IP allowlists when possible
  • Use HTTPS for all API calls
  • Respect RBAC and tenant scoping

Operational safeguards

APIs enforce strict validation, rate limiting, and throttling to prevent abuse and maintain platform stability.

Audit integrity

All API calls—including failures—are logged with timestamp, actor, and request metadata for compliance review.
