Access Token Lifecycle

How Relancy manages access tokens using short-lived credentials, refresh workflows, and revocation rules.

Lifecycle Overview

Access tokens authenticate user actions within Relancy. Tokens are short-lived, automatically refreshed, and revoked when security conditions require it.

Lifecycle stages

  • Token issuance
  • Token validation
  • Token refresh
  • Token expiration
  • Token revocation

Operational safeguards

Tokens are bound to device and session metadata. Compromised tokens are invalidated immediately across all services.

Audit integrity

Token lifecycle events include timestamp, identity provider metadata, and revocation reasons for compliance review.
