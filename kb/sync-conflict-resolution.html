Conflict Overview

Sync conflicts occur when offline changes overlap with updates made by other users. Relancy resolves conflicts using deterministic rules to maintain data integrity.

Conflict types

  • Field‑level conflicts
  • Record‑level conflicts
  • Attachment conflicts
  • Lifecycle conflicts

Resolution rules

  • Server‑side authoritative for protected fields
  • Most recent timestamp wins for standard fields
  • Offline drafts preserved when possible
  • Conflicts logged for administrative review

Audit integrity

All conflicts and resolutions are logged with full detail for compliance and operational review.
