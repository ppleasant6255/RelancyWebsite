Data Import and Synchronization

How to bring employee and operational data into Relancy safely and accurately.

Back to Knowledge Base

Overview

Relancy supports structured data import and synchronization workflows to help organizations onboard employees, update records, and maintain accurate operational data. All imports are validated, logged, and tenant‑scoped.

Supported import types

  • Employee profiles
  • Department and hierarchy structures
  • Training assignments
  • Historical training completions
  • Location and site data

Import methods

  • CSV upload: For one‑time or periodic imports.
  • API ingestion: For automated synchronization from HRIS or LMS systems.
  • Webhook‑driven updates: For event‑based synchronization.

Validation rules

All imports are validated for:

  • Required fields
  • Duplicate records
  • Invalid formats
  • Conflicting identifiers

Audit logging

Every import is logged with timestamp, actor, and summary of changes. Errors are recorded for review.

Synchronization best practices

  • Use stable employee identifiers.
  • Automate updates through API where possible.
  • Review validation errors after each import.
  • Keep HRIS as the system of record.
