App Update and Versioning Behavior

How Relancy handles app updates, version compatibility, and feature rollout sequencing.

Versioning Overview

Relancy uses semantic versioning to track app updates and ensure compatibility across modules. Updates may include new features, bug fixes, or security enhancements.

Update behavior

  • Automatic updates via app stores
  • Backward‑compatible API changes
  • Graceful handling of deprecated features
  • Feature flags for controlled rollout

Compatibility rules

The app remains compatible with the current API version and may support limited backward compatibility for transitional periods.

Operational impact

Updates do not affect tenant data or audit history. Users may continue working during background updates.
