Pagination Overview

Relancy APIs use cursor‑based pagination to ensure efficient, scalable data retrieval. Pagination prevents performance issues when querying large datasets.

Filtering options

  • Date ranges
  • Category and severity
  • Location and department
  • Assigned owner
  • Status and lifecycle stage

Sorting rules

APIs support sorting by timestamp, severity, category, or custom fields depending on entity type.

RBAC enforcement

Filters and pagination respect RBAC boundaries. Unauthorized records are excluded automatically.

Performance considerations

Cursor‑based pagination ensures consistent performance even as datasets grow over time.
