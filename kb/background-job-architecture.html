Background Job Architecture

How Relancy processes asynchronous tasks using distributed workers, queues, and retry logic.

Back to Knowledge Base

Architecture Overview

Background jobs handle asynchronous tasks such as notifications, integrations, analytics, and cleanup operations. Jobs run in distributed workers with tenant isolation.

Job categories

  • Notification delivery
  • Webhook dispatch
  • Analytics processing
  • Data cleanup
  • Integration sync tasks

Operational safeguards

Jobs include retry logic, dead-letter queues, and rate limiting. Failures trigger alerts and diagnostic logging.

Audit integrity

Job execution events include timestamp, worker identity, payload metadata, and retry history.
