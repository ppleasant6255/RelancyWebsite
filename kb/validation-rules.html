Validation Rules: Titles, Descriptions, and Deletion Reasons

Relancy Platform Knowledge Base

1. Purpose

This article defines the minimum validation requirements for record titles, descriptions, and deletion reasons across all CaseBunker modules. These rules ensure consistency, clarity, and audit integrity throughout the platform.

2. Validation Overview

All user‑submitted text fields must meet minimum length requirements and cannot be empty. Validation is enforced at both the client and server layers to maintain data quality and prevent incomplete or ambiguous records.

3. Record Title Requirements

  • Minimum Length: 3 characters
  • Required: Yes

Validation Error Messages

  • Missing: “Title is required.”
  • Too Short: “Title must be at least 3 characters.”

4. Record Description Requirements

  • Minimum Length: 10 characters
  • Required: Yes

Validation Error Messages

  • Missing: “Description is required.”
  • Too Short: “Description must be at least 10 characters.”

5. Deletion Reason Requirements

Deletion is a tenant‑governance action. Only Tenant Admins may delete records, and every deletion must include a justification. This requirement supports auditability and prevents untraceable data removal.

  • Minimum Length: Recommended 5–10 characters
  • Required: Always

Validation Error Messages

  • Missing: “A reason is required to delete this record.”
  • Too Short: “Deletion reason must be more detailed.”

6. Enforcement Model

  • Validation is enforced on all create, update, and delete operations.
  • Client‑side validation prevents accidental submission.
  • Server‑side validation ensures integrity regardless of client behavior.
  • All validation failures return structured error messages for UI display.

7. Audit Considerations

All validation failures are non‑destructive and do not modify the record. Deletion reasons are stored in the audit log alongside the actor, timestamp, and record metadata. Soft‑delete is applied by default; hard‑delete is reserved for Relancy Global Admins under legal or compliance directives.
