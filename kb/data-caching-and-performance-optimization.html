Caching Overview

Relancy uses strategic caching to reduce load times, minimize network usage, and improve responsiveness across the app. Cached data is tenant‑scoped and respects RBAC boundaries.

Cache types

  • Local device cache for recently accessed records
  • In‑memory session cache for navigation‑level performance
  • Prefetching of dashboard and list data

Cache invalidation

Relancy automatically invalidates cached data when updates occur, ensuring users always see accurate and up‑to‑date information.

Performance optimizations

  • Lazy loading of large lists
  • Incremental sync for offline mode
  • Optimized API pagination
  • Compression of large payloads

Security considerations

Sensitive or restricted data is never cached on device unless explicitly allowed by policy.
