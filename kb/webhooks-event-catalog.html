Webhooks: Event Catalog and Usage Guide

How to receive real-time notifications from Relancy.

Back to Knowledge Base

Overview

Webhooks allow your systems to receive real-time notifications when important events occur inside Relancy. They are the primary outbound integration mechanism in the SaaS environment and are designed to be simple, reliable, and secure.

How webhooks work

When a subscribed event occurs, Relancy sends an HTTPS POST request to your configured endpoint. The payload includes:

  • Event type
  • Timestamp
  • Tenant identifier
  • Entity reference
  • Relevant metadata

Event catalog

Common webhook events include:

Case events

  • case.created
  • case.updated
  • case.escalated
  • case.closed

Incident events

  • incident.reported
  • incident.validated
  • incident.resolved

Investigation events

  • investigation.opened
  • investigation.findings.published
  • investigation.closed

Training events

  • training.assigned
  • training.completed
  • training.overdue

Security and verification

Webhook requests include a signature header that your system can verify to ensure authenticity. HTTPS is required for all webhook endpoints.

Best practices

  • Respond with HTTP 200 as quickly as possible.
  • Perform heavy processing asynchronously.
  • Use retries to handle transient failures.
  • Log webhook deliveries for audit and troubleshooting.
