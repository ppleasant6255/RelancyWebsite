API Authentication Models

How Relancy secures access to its read-only and event-driven APIs.

Back to Knowledge Base

Overview

Relancy’s Integration API uses secure, tenant-scoped authentication models to ensure that only authorized systems can access your organization’s data. This article explains the supported authentication mechanisms and how they protect your environment.

API keys

API keys provide a simple, secure way for trusted systems to authenticate. Keys are:

  • Tenant-scoped
  • Revocable at any time
  • Rotatable without downtime

Service principals

For enterprise integrations, Relancy supports service-principal authentication. This model provides:

  • Fine-grained access control
  • Credential rotation policies
  • Audit logging of all access

Webhook signature verification

Outbound webhook requests include a signature header that your system can verify to ensure authenticity. HTTPS is required for all webhook endpoints.

Rate limiting and safety

All API access is subject to rate limiting to protect the platform and ensure fair usage across tenants.
