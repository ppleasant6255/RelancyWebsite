Understanding the Integration API

Safe, predictable ways to connect external systems to Relancy.

Back to Knowledge Base

Overview

The Relancy Integration API is designed to give tenants practical integration options without exposing the core operational API that powers the platform. It focuses on read-only access, event notifications, and safe extension points rather than write-level control.

Design principles

  • Tenant-scoped: Every call is bound to a specific tenant.
  • Audit-friendly: All interactions are traceable.
  • Read-first: Reporting and exports are prioritized over mutation.
  • Event-driven: Webhooks notify external systems of important changes.
  • Safe by default: No ability to bypass lifecycle or platform rules.

What the Integration API supports

Outbound: Relancy → your systems

Relancy can notify your systems when important events occur, such as:

  • Case or incident created
  • Case escalated or closed
  • Training assigned, completed, or overdue
  • Pattern or risk signal detected

These notifications are typically delivered via webhooks, allowing your systems to react without polling.

Read-only access

The Integration API and reporting surfaces allow you to pull data from Relancy into your own BI tools, data warehouses, or internal dashboards. This includes:

  • Case and incident data
  • Training and readiness records
  • Relevant operational metrics

What the Integration API does not allow

To protect audit integrity and lifecycle discipline, the Integration API does not allow:

  • Arbitrary write-level access to core entities
  • Custom logic injection into the platform
  • Bypassing lifecycle stages or approvals
  • Modifying training lineage or historical records

Organizations that require this level of control should consider the Private Deployment tier, where dedicated infrastructure and optional write-level APIs are available.

Authentication and security

Access to the Integration API is authenticated and tenant-scoped. Depending on your organization’s needs, this may involve API keys, service principals, or other secure mechanisms. All requests are subject to authorization checks and rate limiting to protect the platform and other tenants.
