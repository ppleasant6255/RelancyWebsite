Framework Overview

The Data Classification Framework categorizes data into sensitivity levels to ensure appropriate handling, storage, and access controls across the platform.

Classification levels

  • Public – Non‑sensitive, tenant‑approved content
  • Internal – Operational metadata and non‑sensitive records
  • Confidential – Employee data, incidents, cases
  • Restricted – Protected‑class data, legal matters, sensitive HR/Security content

Handling rules

  • Restricted data requires elevated roles
  • Confidential data is encrypted and access‑controlled
  • Internal data follows standard RBAC
  • Public data may be shared externally with approval

Audit integrity

Classification changes and access events are logged for compliance and defensibility.
