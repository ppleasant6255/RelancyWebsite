Tenant Onboarding and Provisioning

How new tenants are created, configured, and activated in Relancy.

Back to Knowledge Base

Overview

Tenant onboarding is the process of creating and configuring a new Relancy environment. This includes provisioning isolated data storage, applying default policies, and preparing the tenant for operational use.

Provisioning steps

  • Create isolated tenant database
  • Initialize storage and encryption keys
  • Apply default lifecycle and audit policies
  • Configure role-based access templates
  • Enable reporting and webhook endpoints

Initial configuration

After provisioning, tenants complete initial setup tasks such as:

  • Adding users and roles
  • Configuring training requirements
  • Setting up webhook endpoints
  • Importing initial employee data

Onboarding timeline

Most tenants can begin using Relancy immediately after provisioning. Additional configuration steps can be completed at their own pace.

Private Deployment onboarding

Private Deployment tenants follow a coordinated onboarding process that includes infrastructure setup, custom SLAs, and optional logic extensions.
