API Rate Limits and Fair‑Use Policy

How Relancy ensures stable, fair API access across all tenants.

Back to Knowledge Base

Overview

API rate limits protect platform stability and ensure fair access for all tenants. Relancy enforces rate limits on read‑only and webhook endpoints to prevent abuse or accidental overload.

Rate limit categories

  • Standard API requests
  • Webhook delivery retries
  • Bulk export operations

Fair‑use expectations

  • Use caching where appropriate
  • Avoid unnecessary polling
  • Use webhooks for event‑driven workflows

Limit notifications

Applications may receive rate‑limit responses when thresholds are exceeded. These events are logged for review.

Private Deployment

Private Deployment tenants may configure custom rate limits based on infrastructure capacity.
