Reporting and Data Export Options

How to access and extract your organization’s data from Relancy.

Back to Knowledge Base

Overview

Relancy provides multiple reporting and export options to help organizations analyze trends, support investigations, and meet compliance requirements. All reporting surfaces are read-only to preserve audit integrity and lifecycle discipline.

Built-in reports

Relancy includes a library of built-in reports covering:

  • Case and incident activity
  • Investigation timelines
  • Training completion and readiness
  • Risk signals and patterns
  • Operational performance metrics

These reports can be filtered, sorted, and exported for offline analysis.

CSV exports

Most lists and reports can be exported as CSV files. CSV exports include:

  • All visible fields
  • Custom fields
  • Relevant metadata

CSV exports are ideal for spreadsheets, BI tools, and archival purposes.

Read-only reporting API

The Integration API provides read-only access to key datasets. This allows organizations to:

  • Pull data into BI platforms
  • Automate compliance reporting
  • Build internal dashboards

The reporting API is tenant-scoped and cannot modify platform data.

Data retention and backups

Relancy maintains tenant-specific backups and retention policies. Exported data reflects the current state of your tenant at the time of export.
