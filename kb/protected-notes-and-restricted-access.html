Protected Notes and Restricted Access

How Relancy safeguards highly sensitive case information through restricted note types.

Back to Knowledge Base

Overview

Protected notes allow authorized users to document highly sensitive information that should not be visible to general case participants. These notes are restricted by role and governed by strict audit controls.

When to use protected notes

  • Protected-class disclosures
  • Confidential HR information
  • Legal or compliance-sensitive details
  • Security-sensitive operational data

Access rules

Only users with explicit permissions (e.g., HR Admin, Security Lead) can view or create protected notes.

Audit logging

All protected note activity—including views—is logged in the audit trail with timestamp and actor identity.

Reporting

Protected notes are excluded from general reports and exports unless the user has appropriate permissions.
