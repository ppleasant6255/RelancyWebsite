Error Handling Overview

Relancy uses structured error handling to ensure reliability and clarity. Transient failures trigger automatic retries, while user‑visible errors provide actionable guidance.

Retry logic

  • Automatic retries for network failures
  • Exponential backoff for repeated failures
  • Graceful fallback for offline mode

User‑visible errors

Errors are displayed with clear messaging, avoiding technical jargon and providing next steps when applicable.

Audit and logging

System‑level errors are logged for operational monitoring and platform diagnostics.

Developer considerations

API consumers should implement retry logic and handle rate‑limit responses gracefully.
