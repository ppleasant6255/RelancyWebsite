Propagation Overview

Metadata propagation ensures that updates to employees, locations, departments, categories, and tags are reflected consistently across incidents, cases, training, readiness, and reporting.

Propagation rules

  • Employee updates propagate to all related records
  • Location changes update routing and dashboards
  • Category updates cascade to classification rules
  • Tag updates reflect in reporting and exports
  • Readiness changes update risk models

Operational safeguards

Propagation is atomic, validated, and logged to prevent partial updates or data drift.

Audit integrity

All propagation events include timestamp, actor, affected modules, and before/after values for defensibility.
