Visibility Overview

Role‑based visibility ensures that users only access information appropriate to their responsibilities. Visibility rules apply across incidents, cases, investigations, readiness, and reporting.

Role categories

  • Employees
  • Managers
  • HR roles
  • Security roles
  • Compliance roles
  • Leadership
  • Administrators

Visibility rules

  • Employees see only their own data
  • Managers see direct reports only
  • HR sees HR cases and relevant incidents
  • Security sees security incidents and investigations
  • Leadership sees aggregated, anonymized data
  • Administrators see configuration but not sensitive case content

Audit integrity

All access events are logged, including attempts to view restricted content.
