Browser Session Isolation

How Relancy isolates browser sessions to prevent cross‑user data exposure and ensure secure multi‑session behavior.

Isolation Overview

Browser session isolation ensures that each user session is fully segregated, preventing data leakage across tabs, windows, or shared devices. Isolation is enforced at the token, storage, and cache levels.

Isolation rules

  • Session tokens scoped per tab
  • Restricted local storage usage
  • Automatic cache clearing on logout
  • Blocked cross‑tab session sharing in kiosk mode

Operational impact

Isolation improves security in shared environments and prevents accidental cross‑user visibility.

Audit integrity

Session isolation events and anomalies are logged for compliance and operational review.
