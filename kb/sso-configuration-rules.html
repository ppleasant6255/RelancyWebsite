Configuration Overview

SSO configuration ensures that users authenticate through their organization’s identity provider. Relancy supports multiple SSO protocols and tenant‑specific configuration.

Configuration requirements

  • Valid metadata from identity provider
  • Tenant‑specific redirect URIs
  • Role and group mapping rules
  • MFA enforcement policies

Operational impact

SSO improves security, reduces password fatigue, and centralizes access control.

Audit integrity

All SSO configuration changes are logged with timestamp, actor, and justification for compliance review.
