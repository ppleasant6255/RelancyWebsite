Standards Overview

Relancy follows industry‑standard secure coding practices to prevent vulnerabilities such as injection attacks, insecure deserialization, and cross‑site scripting.

Core principles

  • Input validation and sanitization
  • Least‑privilege access
  • Secure session handling
  • Safe error handling
  • Dependency hygiene
  • Immutable audit logging

Code review requirements

All code changes undergo peer review with security checks. High‑risk modules require senior review.

Audit integrity

Code review logs and security approvals are retained for compliance and operational oversight.
